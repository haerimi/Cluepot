1. 서비스 개요

핵심 가치앱 설치 없이 링크 하나로 위치 취합 → 최적 약속 장소 추천
타겟 사용자스마트폰을 사용하는 20~35세, 2~4명 단위 약속을 자주 잡는 사람
핵심 차별화프라이버시 보호 + 상권 기반 중간지점 + AI 리뷰 분석 + 이탈 방지 온보딩

1.1 두 가지 사용자 역할

구분방장 (Host)참여자 (Guest)
정의약속을 먼저 시작하는 사람링크를 받아 입장하는 사람
진입MeetSpot 직접 접속카카오톡 링크 클릭
권한카테고리 설정, 방 생성, 결과 확정위치 입력, 이동 방식 투표

2. 상세 기능 명세 (Feature List)

2.1 Step 1 — 카테고리 선택

기본 동작
방장이 앱 진입 시 첫 화면에서 카테고리를 선택
선택 가능 카테고리: 맛집 / 카페 / 술집 / 브런치 / 디저트
카테고리는 Kakao Local API의 category_group_code와 매핑
카테고리Kakao 코드비고
맛집FD6음식점 전체
카페CE7카페, 디저트 포함
술집FD6 + 키워드 필터'술집', '바', '포차' 키워드 추가 필터링
브런치FD6 + 키워드 필터'브런치', '샌드위치' 키워드 추가 필터링
디저트CE7 + 키워드 필터'케이크', '마카롱' 키워드 추가 필터링

2.2 Step 2 — 위치 입력 & 프라이버시 보호

기본 동작
방장: 카테고리 선택 후 본인 출발지 입력
참여자: 링크 진입 후 본인 출발지 입력
핵심 설계 원칙: 상세 주소 미노출
사용자가 '서울 마포구 연남동 239-1'을 입력해도, 시스템은 가장 가까운 지하철역(홍대입구역) 또는 법정동(연남동)으로 추상화하여 저장하고 표시합니다. 상대방에게는 추상화된 위치만 공개됩니다.

위치 추상화 로직

1. 사용자 입력값 → Kakao 주소 검색 API로 위경도(lat, lng) 획득
2. 해당 좌표 기준 반경 500m 내 지하철역 검색 (category_group_code: SW8)
3. 가장 가까운 역이 있으면 → '홍대입구역 근처'로 표시
4. 역이 없으면 → 법정동 이름(예: '연남동')으로 표시

2.3 Step 3 — 링크 공유 & 실시간 동기화

심리적 장치: 예상 소요 시간 노출
링크를 받은 참여자가 진입 즉시 '위치 입력까지 약 20초면 돼요!'라는 메시지와 함께 진행 단계(1/2 위치 입력 → 2/2 완료)를 시각적으로 보여줍니다.

2.4 Step 4 — 이동 방식 선택

상권 중심 중간지점 알고리즘 (Hotspot Optimization)
단순 좌표 평균값은 상권이 없는 공원 한복판이 나올 수 있습니다. MeetSpot은 중간지점 계산 후 실제 상권이 있는 가장 가까운 지하철역 상권으로 자동 보정합니다.

2.5 Step 5 — 장소 추천 & AI 셜록 모드

AI 셜록 모드 — Gemini API 리뷰 분석

태그트리거 키워드 예시의미
#데이트분위기, 조명, 둘이, 연인, 아늑커플 방문에 적합
#회식단체, 넓은, 룸, 시끄러워도, 왁자단체 모임에 적합
#조용한대화조용, 대화, 집중, 카공진지한 대화/업무 적합

4. AI 프롬프트 설계 (Gemini API)

System Prompt

You are a Korean restaurant and cafe review analyst. Your task is to analyze customer reviews and extract: 1. Visit purpose tags: #데이트, #회식, #조용한대화, #혼밥, #가성비, #인스타감성 2. One-line insight in Korean (max 30 characters) Respond ONLY in JSON format: { "tags": ["#데이트", "#조용한대화"], "insight": "조명이 어둡고 조용해서 대화에 집중하기 좋아요" }
