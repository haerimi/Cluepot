# CluePot PRD

## 1. 제품 개요

**제품명**: CluePot
**한 줄 설명**: 참가자 모두에게 공정한 모임 장소를 AI가 추천해주는 모임 조율 플랫폼

**핵심 문제**: 여러 명이 모일 장소를 정할 때 다수결이나 한 명의 판단에 의존하게 되어 이동 부담이 불균등하게 분배된다.

**솔루션**: 참가자 각자의 출발 지역·교통수단·분위기 선호를 수집해 AI(PINI)가 이동 부담이 균등한 최적 장소를 추천한다.

---

## 2. 타겟 사용자

| 페르소나 | 설명 |
| :--- | :--- |
| **호스트** | 모임을 주최하고 장소를 결정하는 사람. 참가자 응답을 기다린 뒤 PINI를 실행한다 |
| **참가자** | 코드로 초대받아 참가하는 사람. 자신의 위치와 이동 방식을 입력하기만 하면 된다 |

---

## 3. 핵심 기능

### 3-1. 모임(Room) 관리
- 호스트가 모임 생성 시 카테고리(음식점·카페·바·브런치·디저트) 선택
- 고유 입장 코드 발급 → 참가자에게 공유
- 모임 이름·사진 수정 가능
- 상태: `waiting` → `voting` → `done`

### 3-2. 참가자 위치 수집
- 참가자 각자 출발 지역(동 단위) + 교통수단(도보·대중교통·자가용·자전거) 입력
- 정확한 주소 불필요, 동 단위로도 동작

### 3-3. PINI Mode (AI 장소 추천)
- 호스트가 실행 → 참가자 전원의 이동 균형 계산
- 이동 시간 최대-최소 차이를 최소화하는 후보군 생성
- 분위기 선호 오버레이 → 광고성 후기 필터링 후 신뢰 점수 계산
- 이유 포함 추천 (균형도 수치 + 참가자별 이동 시간 공개)

### 3-4. 일정(Schedule) 확정
- 호스트가 추천 장소 선택 → 일정 확정
- 확정된 일정은 모든 참가자의 캘린더에 반영
- 일정 상세: 장소명·주소·날짜·메모

### 3-5. 프로필
- 닉네임·프로필 이미지 설정
- 이미지 업로드 → Supabase Storage 저장
- 모임 참가자 카드, 네비게이션 아바타에 반영

### 3-6. 인증
- 이메일/비밀번호 회원가입·로그인 (Supabase Auth)
- 로그인 필요 페이지: `/calendar`, `/rooms`, `/profile`

---

## 4. 사용자 플로우

```
[호스트]
랜딩 → 모임 만들기 → 카테고리 선택 → 코드 발급
  → 참가자 응답 대기 → PINI 실행 → 장소 선택 → 일정 확정

[참가자]
코드로 참가 → 위치·교통수단 입력 → 호스트 결정 대기 → 일정 확인
```

## 7. 비기능 요구사항

- **프라이버시**: 위치 데이터는 분석 후 저장하지 않음
- **모바일 퍼스트**: 주요 플로우는 모바일(430px) 기준 설계
- **접근성**: 모임 코드 공유만으로 참가 가능 (앱 설치 불필요)
